# v0.14.0 — MAST-Evidence Dispatch Hardening

**Released:** 2026-05-26
**Theme:** Dispatch-brief and orchestration hardening driven by empirical multi-agent failure data — MAST taxonomy (arXiv 2503.13657, NeurIPS 2025), 1,600+ production traces across 5 systems.

## TL;DR

Four changes to how the Master Orchestrator dispatches and tracks specialist work, each tied to a named, measured multi-agent failure mode instead of intuition:

| Change | Failure mode addressed | Frequency in MAST data |
|---|---|---|
| `DONE WHEN:` field | FM-1.5 unaware of termination | 12.4% |
| `PRIOR CONTEXT` → `COMPLETED:`/`OPEN:` split | FM-1.3 step repetition (#1 failure mode) | 15.7% |
| Specialist Registry `Default Tier` column | cost, not a MAST failure mode | 50-80% projected cost cut |
| Refinement Counter | "coherence trap" (Reflexion ICLR 2024) | not in MAST, separate evidence line |

Plus: Rule 8 reframed from a negative ("Explain = Hallucinate") to a positive instruction, per Pink Elephant Problem evidence that LLMs underperform on negated instructions.

## Why this release

A synthesis pass across 5 parallel research agents (2026-05-26) pulled 80+ sources: Anthropic official docs, arXiv 2025-2026 papers, and production retrospectives from Shopify, Stripe, Airbnb, GitHub Copilot, and Cursor. Two data points stood out:

- **MAST taxonomy** (NeurIPS 2025) measured failure rates of 41-86.7% across 1,600+ multi-agent traces. The top two failure modes — step repetition (15.7%) and unawareness of termination conditions (12.4%) — mapped directly onto gaps in Kasidit's dispatch brief format at the time.
- **SWE-Bench Verified vs Pro gap**: the same Claude Opus scored 80.9% on Verified vs 45.9% on Pro — a 35-point gap attributed to training contamination on Verified. Pro tasks average 4.1 files modified, 107 lines changed; context overflow was identified as the #1 blocker. This reinforced that dispatch hygiene (not raw model capability) is where the framework can move the needle.

## What's new

### 1. `DONE WHEN:` field in the dispatch brief

Every specialist dispatch must now declare measurable completion signals:

```
MISSION: <one sentence>
INPUTS: <files, versions>
CONSTRAINTS: <deadline, compat>
EXPECTED OUTPUT: <agent's documented output>
DONE WHEN: <measurable completion signals — tests pass, no lint, screenshot matches>
PRIOR CONTEXT: <see #2>
```

Addresses MAST FM-1.5 (12.4% of multi-agent failures) — agents that don't know when to stop tend to either under-deliver or keep "polishing" past the point of usefulness.

### 2. `PRIOR CONTEXT` split into `COMPLETED:` + `OPEN:` with `[agent-name]` attribution

```
PRIOR CONTEXT:
  COMPLETED:
    [deep-researcher #1] HeyGen API: audio-driven mode confirmed via official docs
    [architect-planner] folder tree finalized, state machine spec written
  OPEN:
    Implement Day 1 scaffold per architect spec.
```

Addresses MAST FM-1.3 — step repetition, the single most frequent multi-agent failure mode (15.7%). Un-attributed shared state was the direct cause: a specialist re-summarizing "what's been done" in prose, rather than reading an explicit completed-list, would sometimes redo finished work. The `[agent-name]` prefix also targets FM-2.6 (reasoning-action mismatch, 13.2%) by making it traceable which agent produced which fact.

### 3. Specialist Agent Registry — `Default Tier` column

Every agent now carries an explicit model tier assignment:

| Category | Default tier |
|---|---|
| Read-only research (`deep-researcher`, `legacy-specialist` read paths) | Haiku |
| Analytical/synthesis (`bug-hunter`, `audit-specialist`, `test-writer`, `refactor-surgeon`) | Sonnet |
| Creative/high-stakes (`migration-specialist`, `architect-planner`) | Sonnet, escalates to Opus on demand |

Specific changes from the prior all-Sonnet default:
- `deep-researcher`: sonnet → **haiku** (3× cheaper for read-only paths)
- `architect-planner`: opus → **sonnet** (was over-provisioned — Sonnet handles plan-only work fine)
- `legacy-specialist`: → **haiku** explicit for read-only paths

Includes a 20% correction-rate watch — if a Haiku-tier agent's re-prompt rate crosses 20%, the savings are negated and the mission escalates to Sonnet. Projected cost reduction on read-heavy multi-agent runs: **50-80%**. Example: a typical `/kasi-multi 6` dispatch (3 researchers + 2 reviewers + 1 architect) drops from ~$0.45 to ~$0.23.

### 4. Refinement Counter (separate from the Failure Counter)

```
Mission Counter (existing) — fail = runtime did not pass
Refinement Counter (new) — same approach attempted with minor tweak
  - max 3 refinements on the same step
  - if confidence label is the same or lower for 2 turns in a row → halt, escalate to user
```

Evidence: Reflexion (ICLR 2024) found that intrinsic self-correction without an external signal degrades performance over iterations — the "coherence trap" of increasingly polished but still-wrong output. The counter is a safety addition; it doesn't change behavior on the success path.

### 5. Rule 8 reframed positive

| Before | After |
|---|---|
| "Explain = Hallucinate" | "Output direct, reserve explanation" |

Evidence: the Pink Elephant Problem (arXiv 2503.22395, 2025) confirms LLMs "fail to capture lexical semantics of negation" — telling a model what *not* to do is measurably less effective than telling it what *to* do. Behavior is identical; only the framing changed, on Rule 8 specifically. Hard-safety prohibitions (destructive ops, auto-commit) were deliberately **not** converted — negative framing stays where the cost of a miss is high.
