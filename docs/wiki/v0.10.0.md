# v0.10.0 — Honesty Cleanup

> Released: 2026-04-26
> Theme: **Honesty over polish.** Merge SKILL.md split, consolidate audit agents, mark spec vs runtime clearly.

## Headline changes

- **Full Framework merged back into `SKILL.md`** behind a prompt-level Mode gate (`off` / `router` / `lite` / `full` / `ultra`). The earlier v0.9.x `SKILL-full.md` split was reverted — it created two sources of truth that drifted. One file, mode-gated load depth.
- **`audit-specialist` agent** consolidates `code-reviewer` / `security-auditor` / `perf-profiler` via `--focus=quality|security|perf|all`. Old agents remain as **name-recognition stubs** (no automatic router mapping — users must invoke `audit-specialist` explicitly). Stubs scheduled for removal in v0.11.
- **`/kasi-init` install prompt** clarified — digit-only input. Anything not `1/2/3` falls back to `1` (router). No silent guessing.
- **`/kasi` mode precedence marked as spec, not runtime.** No code in this plugin reads the three config files (session > project > global > default), merges them, and returns a single resolved mode. The chain is documentation of intended priority — applied by the user and the AI reading the config files directly. Treat the table in `commands/kasi.md` as a contract, not an enforcer.
- **Backend hooks (runtime-enforced)** for the first time:
  - `kasidit-route.py` (UserPromptSubmit) — prompt classifier + memory query
  - `kasidit-verify.py` (PostToolUse + Stop) — confidence-label + orchestrator-violation check
  - `kasidit-record.py` (Stop) — parses `[kasidit-log|pattern|memory|rule]` emit lines into JSONL
  - `kasidit-update-check.sh` (SessionStart) — 1×/day release-tag check
  - `kasidit-drift-check.sh` (SessionStart) — Centerlite-sync reminder
- **`install.sh`** canonical installer — copies hooks, merges `~/.claude/settings.json`, seeds Gravity hub, installs default checklists. Idempotent. Backs up settings before merging.
- **12 default checklists** ship under `defaults/checklists/` — `code-review`, `security`, `perf` × `php / node / python / go`. Seeded into `~/.claude/skills/kasidit/center/checklists/` at install time. Project `.kasidit/CHECKLISTS/` may copy and customize.
- **Incremental backend save** — AI emits `[kasidit-log] kind=... mode=... turns=... outcome=...` lines at mission end; `kasidit-record.py` appends to `route-memory.jsonl`. Router learns shortest successful route per mission kind over time.
- **`sudo <mission>`** clarified — parallel fan-out, **min 2 agents**, assumption-narrated pacing. **Not** a permission escalation. `/kasi-multi --fast` is an equivalent alternative.

## Why "Honesty Cleanup"

Three things drifted in v0.9.x:

1. **Two SKILL files** — `SKILL.md` and `SKILL-full.md` slowly diverged. Users loaded one or the other and got different behavior with the same plugin version.
2. **Three audit agents with overlapping scope** — `code-reviewer`, `security-auditor`, `perf-profiler` shared 60% of their checklists. Calling the wrong one wasted tokens and split memory.
3. **Spec mistaken for runtime** — `/kasi`'s precedence table read like enforcement but no code resolved it. The README implied a runtime layer the plugin did not have.

v0.10 fixes by collapsing duplicates and labeling spec-vs-runtime explicitly.

## Migration notes

### From v0.9.2

- Existing `~/.claude/skills/kasidit/SKILL.md` is **not** auto-replaced by `install.sh` — it copies hooks + seeds checklists only. Update SKILL.md manually:
  ```
  cp ~/.claude/plugins/marketplaces/kasidit/plugins/kasidit/skills/kasidit/SKILL.md \
     ~/.claude/skills/kasidit/SKILL.md
  ```
- Old `code-reviewer` / `security-auditor` / `perf-profiler` invocations still work (stubs delegate to `audit-specialist --focus=...`). Update your call sites by v0.11 — stubs disappear then.
- Hooks v0.9.2 (`kasidit-log.{sh,py}`) coexist with v0.10 hooks. `install.sh` does not remove them.

### Settings.json schema

`install.sh` registers 6 hook events:

```json
{
  "hooks": {
    "SessionStart":    [{ "type": "command", "command": "bash ~/.claude/hooks/kasidit-update-check.sh" },
                        { "type": "command", "command": "bash ~/.claude/hooks/kasidit-drift-check.sh" }],
    "UserPromptSubmit":[{ "type": "command", "command": "python3 ~/.claude/hooks/kasidit-route.py" }],
    "PostToolUse":     [{ "type": "command", "command": "python3 ~/.claude/hooks/kasidit-verify.py" }],
    "Stop":            [{ "type": "command", "command": "python3 ~/.claude/hooks/kasidit-verify.py" },
                        { "type": "command", "command": "python3 ~/.claude/hooks/kasidit-record.py" }]
  }
}
```

Backed up to `settings.json.kasidit-backup-<ts>` before merge.

## Known limitations

- **No runtime config resolver.** `/kasi` mode files (session > project > global) describe intended priority. AI behavior matches the contract, but no code enforces a single resolved value at runtime.
- **Old standalone `kasi-*` commands** (without the `kasidit:` namespace) may exist from earlier installs at `~/.claude/commands/`. v0.10 ships only namespaced `kasidit:kasi-*`. Standalone copies will not be updated by `install.sh` — clean them manually if drift becomes a problem.
- **`SKILL.md` not auto-synced** to `~/.claude/skills/kasidit/SKILL.md`. Manual copy required after upgrade (see Migration above).
